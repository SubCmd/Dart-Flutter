# Dart와 Flutter 소개

## Dart란?
> Google이 2011년에 발표한 객체지향 프로그래밍 언어 (Dart3은 2023년 발표)
> 처음에는 JavaScript를 대체할 웹 언어로 만들어졌지만, 현재는 주로 Flutter의 공식 언어로 자리 잡음.

### 1. Dart의 주요 특징?
1. 타입 안정성(Type-Safe): 컴파일 시점에 타입 오류를 잡아줘서 안정적
   Python의 동적 타이핑에 익숙하다면 처음엔 조금 엄격하게 느껴질 수 있지만, 대규모 앱에서 큰 장점
2. 두 가지 컴파일 방식:
   1) JIT(Just-In-Time): 개발 중 핫 리로드(Hot Reload)를 가능하게 해서 코드 변경을 즉시 반영
                        단점: 처음 시작할 때는 컴파일하는 시간이 필요해서 앱 로딩 시간이 다소 걸림.
   2) AOT(Ahead-Of-Time): 배포 시 네이티브 코드로 컴파일되어 빠른 실행 속도를 제공
3. Null Safety: Null 참조 오류를 컴파일 단계에서 방지해줌.
4. 문법이 친숙함: JavaScript, Java, Python을 해본 사람이라면 비교적 빠르게 배울 수 있음.

```dart
// 간단한 Dart 예시
void main() {
    String name = 'Anki Clone';
    print('Welcome to $name!');
}
```

#### Python vs. Dart

| 항목 | Python | Dart / Flutter |
|---|---|---|
| 타입 시스템 | 동적 타이핑 (런타임에 결정, 힌트는 선택) | **정적 + 건전한(sound) 타이핑** (컴파일 타임 검증) |
| 실행 방식 | 인터프리터 | 개발 중 **JIT**(Hot Reload용), 배포 시 **AOT**(네이티브 컴파일) |
| Null 처리 | 모든 것이 `None` 가능 | **Null Safety** — nullable(`?`)을 명시해야만 null 허용 |
| 패키지 매니저 | `pip` + `requirements.txt` / `pyproject.toml` | `pub` + `pubspec.yaml` |
| 가상환경 | `venv` / `conda` (수동 격리) | 프로젝트별 의존성 자동 격리 (`.dart_tool/`) |
| 진입점 | 파일을 직접 실행 (`if __name__ == "__main__"`) | `main()` 함수가 **필수** 진입점 |
| 주 용도 | 백엔드/데이터/스크립팅 | **선언형 UI 앱**(모바일/웹/데스크톱) + CLI |


### 2. AOT vs. JIT - Flutter Hot Reload의 비밀

┌─────────────────────────────────────────────────────┐
│  개발 시 (Development)                               │
│  ─────────────────────                              │
│  소스 코드 ──[JIT 컴파일]──> 즉시 실행                │
│                              ↑                      │
│                              Hot Reload 가능         │
├─────────────────────────────────────────────────────┤
│  배포 시 (Release / Production)                      │
│  ─────────────────────────────                      │
│  소스 코드 ──[AOT 컴파일]──> 네이티브 바이너리         │
│                              ↑                      │
│                              빠른 실행 속도          │
└─────────────────────────────────────────────────────┘

#### Dart의 두 가지 컴파일 모드 (Hot Reload 비밀)

```
[개발 중]  소스코드 ──JIT──▶ 즉시 실행 + 상태 유지한 채 코드 교체 = Hot Reload
[배포 시]  소스코드 ──AOT──▶ 네이티브 기계어 (빠른 시작, 작은 앱 크기)
```
 
- **JIT (Just-In-Time)**: 개발 중. 코드를 바꾸면 앱을 끄지 않고 화면만 갱신 → 생산성의 핵심.
- **AOT (Ahead-Of-Time)**: 배포 빌드. Python처럼 매번 인터프리트하지 않고 미리 네이티브로 컴파일.

## Flutter란?
> Google이 만든 크로스 플랫폼 UI 프레임워크
> 하나의 코드베이스로 iOS, Android, Web, Windows, macOS, Linux 앱을 동시에 만들 수 있음.

### Flutter의 핵심 개념
1. 모든 것이 위젯(Widget): 버튼, 텍스트, 레이아웃, 심지어 패딩까지 전부 위젯
   위젯을 트리 구조로 조합해서 UI를 만듦.
2. 자체 랜더링 엔진(Skia/Impeller): OS의 네이티브 UI 컴포넌트를 사용하지 않고, Flutter가 직접 화면을 그려서 모든 플랫폼에서 동일한 디자인을 보장
3. Hot Reload: 코드를 수정하면 앱 상태를 유지한 채 UI가 즉시 갱신되어 개발 속도가 매우 빠름
4. 풍부한 위젯 라이브러리: Material Design(Android 스타일)과 Cupertino(iOS 스타일) 위젯을 모두 기본 제공

```dart
// 간단한 Flutter 위젯 예시
class HelloCard extends StatelessWidget {
    @override
    Widget build(BuildContext context) {
        return Card(
            child: Text("암기 카드 앞면'),
        );
    }
}
```

## Anki 같은 자기계발 앱 만들기에 어떤지?
1. Cross Platform: 모바일, 데스크탑 동기화 가능 (Flutter는 iOS, Android, Web 한 번에 커버 가능)
2. 부드러운 애니메이션: 60fps 애니메이션 쉽게 구현 가능
3. 로컬 DB 지원: 카드 데이터를 저장할 때, `sqflite`, `isar`, `drift` 같은 라이브러리로 SQLite를 쉽게 다룰 수 있음.
4. 상태 관리 도구의 성숙도: `Riverpod`, `Bloc`, `Provider` 같은 도구로 학습 진행 상태, 카드 덱, 통계 등을 체계적으로 관리 가능

#### 용어 정리
- 핫 리로드(Hot Reload): 개발 중인 애플리케이션을 처음부터 다시 시작할 필요 없이, 소스 코드의 변경 사항을 즉시 적용하는 기능
> 상태유지: 화면 입력값이나 데이터 등 현재의 앱 상태를 그대로 유지하면서 수정된 코드만 반영
> 빠른 피드백: 코드를 저장하는 즉시 실행 화면에 변경 사항이 나타나 디자인 다듬기 등 유리

- 네이티브 코드(Native Code): 특정 운영 체제(OS) 및 하드웨어(CPU) 환경에서 직접 실행될 수 있도록 컴파일된 기계어 코드
> 중간 번역 과정이 없어 실행 속도가 매우 빠르고 성능이 극대화됨
