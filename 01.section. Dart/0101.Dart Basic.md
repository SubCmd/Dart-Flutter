# Phase 1-1: Dart 소개 및 개발 환경

## 1. 개념 및 정의

### 1. Dart란 무엇인가?
Dart는 2011년 Google이 발표한 객체 지향, 클래스 기반, 가비지 컬렉션 언어
처음에는 JavaScript의 대체재로 출발했지만, 현재는 Flutter의 공식 언어로 자리잡음.

| 항목 | 내용 | 
| ---- | ---- | 
| 최신 메이저 | Dart3 (2023년) | 
| 패러다임 | 객제지향 + 함수형 일부 지원 | 
| 타입 시스템 | Sound Static Type System (런타임 타입 안전 보장) |
| Null 처리 | Sound Null Safety |
| 컴파일 | AOT(네이티브 빌드) + JIT(개발 중) |
| 메모리 관리 | 가비지 컬렉션 | 


### 2. AOT vs JIT - Flutter Hot Reload의 비밀

```
┌─────────────────────────────────────────────────────┐
│  개발 시 (Development)                               │
│  ─────────────────────                              │
│  소스 코드 ──[JIT 컴파일]──> 즉시 실행                │
│                              ↑                      │
│                              Hot Reload 가능         │
├─────────────────────────────────────────────────────┤
│  배포 시 (Release / Production)                      │
│  ─────────────────────────────                      │
│  소스 코드 ──[AOT 컴파일]──> 네이티브 바이너리         │
│                              ↑                      │
│                              빠른 실행 속도          │
└─────────────────────────────────────────────────────┘
```

- **JIT(Just-In-Time)**: 런타임에 컴파일 → 코드 수정 시 앱 재시작 없이 변경사항 반영 (Hot Reload)
- **AOT(Ahead-Of-Time)**: 배포 시 미리 네이티브 코드로 컴파일 → 부팅 속도 빠름, 런타임 오버헤드 적음

이 두 가지를 모두 지원하기 때문에 Flutter는 개발 생산성과 배포 성능을 동시에 얻을 수 있음.


### 3. Dart vs. Python vs. JavaScript

| 개념 | Python | JavaScript | Dart |
|------|--------|------------|------|
| 변수 선언 | `x = 5` | `let x = 5;` | `var x = 5;` |
| 상수 | `X: Final = 5` | `const X = 5;` | `const x = 5;` |
| 함수 | `def f(x): ...` | `function f(x) {}` | `int f(int x) {}` |
| 타입 힌트 | optional | optional (TS) | **mandatory by default** |
| Null | `None` | `null` / `undefined` | `null` (with safety) |
| 문자열 보간 | `f"x={x}"` | `` `x=${x}` `` | `'x=$x'` |
| 리스트 | `[1, 2, 3]` | `[1, 2, 3]` | `[1, 2, 3]` |
| 딕셔너리 | `{'a': 1}` | `{a: 1}` | `{'a': 1}` |
| 클래스 진입점 | `if __name__ == '__main__':` | (없음) | `void main() {}` |


### 4. Flutter가 Dart를 선택한 4가지 이유
1) Hot Reload: JIT 덕분에 코드 수정 → 즉시 반영 (1초 미만)
2) 단일 스레드 + 이벤트 루프: UI 랜더링의 일관성과 60fps 보장
3) 타입 안전: 컴파일 타임에 대부분의 버그 차단
4) AOT 컴파일: 배포 시 네이티브 수준의 성능
