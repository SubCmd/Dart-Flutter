## 1-0. 시작점: `main`과 첫 코드

Python은 파일을 그냥 실행하지만, Dart는 `main()` 함수가 반드시 진입점임.

```dart
// hello.dart
void main() {
    print('습관 트래킹 앱을 시작합니다.');
}
```

```bash
dart run hello.dart
```

| Python | Dart |
|---|---|
| `print("hi")` (파일 어디서든) | `print('hi')` (반드시 `main()` 안 또는 호출된 함수 안) |
| `if __name__ == "__main__":` | `void main() { ... }` (이게 곧 진입점) |
| 세미콜론 없음, 들여쓰기로 블록 | **세미콜론 `;` 필수**, `{}`로 블록 |
 
> 💡 처음엔 세미콜론을 자주 빠뜨립니다.
> `dart format`이 잡아주지만, 컴파일러가 에러를 내므로 금방 익숙해집니다.

## 1-1. 변수 선언: `var`/`final`/`const`/`late`

Python은 그냥 `x = 1`이지만, Dart는 "이 변수가 바뀔 수 있는가"를 의도적으로 선택함.

```dart
void main() {
  var count = 0;          // 타입 추론(int). 재할당 가능
  count = 5;              // OK

  final userName = 'yoonseop';  // 런타임 상수. 한 번만 할당 가능
  // userName = 'kim';    // ❌ 컴파일 에러

  const maxRetry = 3;     // 컴파일타임 상수 (고정된 리터럴이어야 함)

  late String token;      // 나중에 초기화하겠다고 약속 (지연 초기화)
  token = fetchToken();   // 사용 전에 반드시 할당해야 함
}
```

| 키워드 | 의미 | Python 대응 (대략) |
|---|---|---|
| `var` | 타입 추론 + 재할당 가능 | 일반 변수 `x = 1` |
| `final` | 한 번만 할당 (런타임에 값 결정) | "재할당 안 하기로 약속한 변수" |
| `const` | 컴파일타임 상수 (불변) | 모듈 상수 `MAX = 3` (진짜 불변) |
| `late` | 선언은 지금, 초기화는 나중에 | 지연 로딩 패턴 |
 
> 💡 **실무 팁**
> 기본값은 `final`을 먼저 쓰고, 바꿔야 할 때만 `var`로 바꾸세요.
> 불변을 기본으로 하면 버그가 줄어듭니다.
> (데이터 분석에서 원본 DataFrame을 안 건드리고 복사본을 쓰는 습관과 같은 맥락)


### `final` vs `const`의 미묘한 차이

```dart
final now = DateTime.now();   // ✅ 런타임에 값이 정해짐
const now2 = DateTime.now();  // ❌ 컴파일타임에 알 수 없으므로 에러
 
const list = [1, 2, 3];       // 내용까지 완전 불변
final list2 = [1, 2, 3];      // 재할당은 불가하지만 내부 요소 변경은 가능
list2.add(4);                 // ✅ OK
```

## 1-3. ⭐ Null Safety — Dart의 핵심 사고방식

Python에서 가장 흔한 런타임 에러는 'AttributeError: 'None Type; object has no attribute ...'
Dart는 **언어 차원에서 원천 차단**


### 기본 규칙: 기본은 null 불가, `?`를 붙여야 null 허용

```dart
String name = 'sub';    // 절대 null이 될 수 없음
name = null;            // ❌ 컴파일 에러

String? nickname;       // null 허용 (자동으로 null로 초기화)
nickname = null;        // ✅ OK
```


### Null 관련 연산자 (반드시 외우기)

```dart
String? userName;
 
// 1) ?. — null이면 호출 안 하고 null 반환 (안전한 접근)
int? len = userName?.length;        // userName이 null이면 len도 null
 
// 2) ?? — null이면 대체값 사용 (Python의 `x or default`와 유사하나 더 엄격)
String display = userName ?? '게스트';
 
// 3) ??= — null일 때만 할당
userName ??= '기본값';
 
// 4) ! — "절대 null 아님" 단언 (위험! 틀리면 런타임 에러)
int length = userName!.length;      // 확신할 때만 사용
```

> 💡 **Python 비유**
> `userName?.length`는 `userName.length if userName is not None else None`을 한 글자로 쓴 것입니다.
> `??`는 `userName or '게스트'`와 비슷하지만, Python의 `or`는 빈 문자열·0도 falsy로 처리하는
> 반면 **Dart의 `??`는 오직 null만** 본다는 점이 다릅니다.

```dart
// 차이 예시
String? a = '';
print(a ?? '대체');   // 출력: (빈 문자열) — ''은 null이 아니므로 그대로
// Python의 (a or '대체')라면 '대체'가 나옴

## 1-4. 문자열 (String)

```dart
void main() {
    var user = 'sub';
    var dau = 1523;

    // 보간(interpolation) - python f-string과 동일한 감
    print('안녕하세요 $user 님');       // 단순 변수
    print('오늘 DAU: ${dau * 2}명')     // 표현식은 ${}

    // 여러 줄 문자열
    var report = '''
    === 일일 리포트 ===
    사용자: $user
    DAU: $dau
    ''';

// 유용한 메서드
print(user.toUpperCase());      // SUB
print(user.contains('su'));     // true
print('   공백   '.trim());     // '공백'
print('a, b, c'.split(','));    // [a, b, c]
}
```

| Python | Dart |
| ------ | ---- |
| `f"{user}"` | `'$user` |
| `f"{x * 2}"` | `'${x * 2}` |
| `"""여러줄"""` | `'''여러줄'''` |
| `str.upper()` | `String.toUpperCase()` |
| `",".join(list)` | `list.join(",")` |

## 1-5. 컬렉션: List / Set / Map

```dart
void main() {
    // List (Python list)
    List<String> habits = ['운동', '독서', '코딩'];
    habits.add('명상');
    print(habits[0]);           // 운동
    print(habits.length);       // 4

    // Set (Python set) - 중복 자동 제거
    Set<String> channels = {'organic', 'paid', 'organic'};
    print(channels);        // {organic, paid}

    // Map (Python dict)
    Map<String, int> metrics = {
        'dau': 1523,
        'wau': 8200,
        'mau': 24000,
    };
    print(metrics['dau']);      //1523
    metrics['retention'] = 42;
}
```


### 컬렉션 안에서 if / for / spread (Python에는 없는 강력한 문법)

```dart
var isPremium = true;

// collection-if & collection-for
var menu = [
  '홈',
  '대시보드',
  if (isPremium) '고급 분석',          // 조건부 요소 추가
  for (var i = 1; i <= 3; i++) '리포트$i',  // 반복 생성
];
// 결과: [홈, 대시보드, 고급 분석, 리포트1, 리포트2, 리포트3]

// spread (...) — 리스트 펼치기
var base = ['a', 'b'];
var extended = [...base, 'c', 'd'];   // [a, b, c, d]
```

> 💡 Python의 리스트 컴프리헨션 `[f(x) for x in items if cond]`
> Dart에서는 **컬렉션 리터럴 안의 `for`/`if`** 또는 **`.map().where()`**(Phase 2)로 표현합니다.

| Python | Dart |
|---|---|
| `[x*2 for x in nums]` | `[for (var x in nums) x * 2]` |
| `[x for x in nums if x > 0]` | `[for (var x in nums) if (x > 0) x]` |
| `[*a, *b]` | `[...a, ...b]` |

## 1-6. 제어문

```dart
void main() {
    var dau = 1523;

    // if / else if / else (조건에 괄호 필수)
    if (dau > 2000) {
        print('목표 초과');
    } else if (dau > 1000) {
        print('순항 중');
    } else {
        print('주의');
    }

    // for - 3가지 형태
    for (var i = 0; i < 3; i++) print(i);       // 전통적
    var habits = ['운동', '독서'];
    for (var h in habits) print(h);             // for-in (Python의 for x in)
    habits.forEach((h) => print(h));            // forEach 메서드

    // while
    var retry = 0;
    while (retry < 3) {
        retry++;
    }
}
```

### Lv.1 - 기초 점검
**문제 1.** 다음 중 컴파일 에러가 나는 줄을 모두 고르고 이유를 쓰세요.

```dart
final name = 'kim';
name = 'lee';                  // (A)
const today = DateTime.now();  // (B)
int dau = null;                // (C)
int? wau = null;               // (D)
```

(A) : ❌ `final`은 런타임 변수로 재할당 불가
(B) : ❌ `DateTime.now()`는 런타임 값이라 `const`(컴파일타임 상수) 불가 → `final`이어야 가능함.
(C) : ❌ `int`는 non-nullable이라 null 불가 → `int?`여야 함
(D) : ✅ 정상 (nullable) - Null safety 기능


**문제 2.** `String? nickname`이 null일 때 `'게스트'`를 쓰고, null이 아니면 그 값을 대문자로 출력하는 코드를 한 줄로 작성하세요.

```dart
print((nickname ?? '게스트').toUpperCase());
// 또는 nickname?.toUpperCase() ?? '게스트' (null이면 '게스트' 그대로)
```
